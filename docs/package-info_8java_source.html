3  * This repository contains a large number of simple coding examples illustrating

10  * \section whatis What is BeepBeep?
11  *
12  * BeepBeep is an event stream processing library. It has been developed by a
13  * team of researchers at <a href="http://liflab.ca">Laboratoire d'informatique
14  * formelle</a>, a research lab located at <a href="http://www.uqac.ca">Université
15  * du Québec à Chicoutimi</a>, Canada.
16  *
17  * There exists extensive and detailed documentation about BeepBeep. This page
18  * only covers cod <em>examples</em> using BeepBeep. To learn more about the tool
19  * itself, please refer to these sources of information:
20  *
21  * <ul>
22  * <li>A complete, 300-page <a href="https://liflab.gitbook.io/event-stream-processing-with-beepbeep-3">textbook</a>
23  * that details the basic concepts and advanced features, hosted on GitBook and also available as
24  * an open-access publication.</li>
25  * <li>BeepBeep's <a href="http://liflab.github.io/beepbeep-3">website</a> for more references,
26  * such as download links and scientific publications</li>
27  * <li>The <a href="http://liflab.github.io/beepbeep-3/javadoc/index.html">API reference</a>
28  * for BeepBeep's core and its <a href="http://liflab.github.io/beepbeep-3-palettes/doc">standard palettes</a>
29  * </ul>
30  *
31  * \section usage How to use this repository
32  *
33  * The source code repository is organized into Java <a href="namespaces.html">packages</a>.
34  * Each package is centered on a particular concept or functionality. For
35  * example, the {@link basic} package contains code examples using BeepBeep's
36  * core processors, while the {@link plots} package shows how to use
37  * processors and functions from the <code>Plots</code> palette.
38  * <p>
39  * Typically, an example is a file that contains a <code>main()</code>
40  * method that you can run. For each file, you can see the detailed
41  * source code directly from this website; this can be done by clicking on
42  * the link that is found at the bottom of each page. See for example the
43  * <a href="_piping_unary_8java_source.html">source code</a> for one of the
44  * simplest examples.
45  *
46  * The source code for all the examples can be downloaded from GitHub at:
47  * <a href="https://github.com/liflab/beepbeep-3-examples"><tt>https://github.com/liflab/beepbeep-3-examples</tt></a>.
48  *
49  * \section run How to run the examples
50  *
51  * The easiest way to run the examples is to open this whole repository into
52  * your favorite IDE, such as <a href="https://eclipse.org/">Eclipse</a> or
53  * <a href="https://netbeans.org/">NetBeans</a>. Simply locate a file with a
54  * <code>main()</code> method, read the associated comments in this file to
55  * understand what happens, and click <i>Run</i>.
56  * <p>
57  * All these examples require at least BeepBeep's core library,
58  * <code>beepbeep-3.jar</code>, in order to run. The latest release of this
59  * library can be <a href="https://github.com/liflab/beepbeep-3/releases">downloaded
60  * from GitHub</a>. Many sections of the repository
61  * focus on the use of one or more <em>palettes</em> and require additional JAR
62  * files. A <a href="https://github.com/liflab/beepbeep-3-palettes/releases">big bundle</a>
63  * of all these files can also be downloaded from GitHub.
64  * <p>
65  * If Ant is installed on your system, typing
66  * <pre>
67  * $ ant download-deps
68  * </pre>
69  * from the root folder of this repository should take care of downloading and
70  * extracting all the required files and place them in the <code>Source/dep</code>
71  * folder.
72  */
