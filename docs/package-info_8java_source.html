19  * \section usage How to use this repository
20  *
21  * The source code repository is organized into Java <a href="namespaces.html">packages</a>.
22  * Each package is centered on a particular concept or functionality. For
23  * example, the {@link basic} package contains code examples using BeepBeep's
24  * core processors, while the {@link plots} package shows how to use
25  * processors and functions from the <code>Plots</code> palette.
26  * <p>
27  * Typically, an example is a file that contains a <code>main()</code>
28  * method that you can run. For each file, you can see the detailed
29  * source code directly from this website; this can be done by clicking on
30  * the link that is found at the bottom of each page. See for example the
31  * <a href="_piping_unary_8java_source.html">source code</a> for one of the
32  * simplest examples.
33  *
34  * \section run How to run the examples
35  *
36  * The easiest way to run the examples is to open this whole repository into
37  * your favorite IDE, such as <a href="https://eclipse.org/">Eclipse</a> or
38  * <a href="https://netbeans.org/">NetBeans</a>. Simply locate a file with a
39  * <code>main()</code> method, read the associated comments in this file to
40  * understand what happens, and click <i>Run</i>.
41  * <p>
42  * All these examples require at least BeepBeep's core library,
43  * <code>beepbeep-3.jar</code>, in order to run. The latest release of this
44  * library can be <a href="https://github.com/liflab/beepbeep-3/releases">downloaded
45  * from GitHub</a>. Many sections of the repository
46  * focus on the use of one or more <em>palettes</em> and require additional JAR
47  * files. A <a href="https://github.com/liflab/beepbeep-3-palettes/releases">big bundle</a>
48  * of all these files can also be downloaded from GitHub.
49  * <p>
50  * If Ant is installed on your system, typing
51  * <pre>
52  * $ ant download-deps
53  * </pre>
54  * from the root folder of this repository should take care of downloading and
55  * extracting all the required files and place them in the <code>Source/dep</code>
56  * folder.
