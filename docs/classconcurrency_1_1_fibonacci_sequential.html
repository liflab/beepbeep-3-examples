Detailed Description

Computes the first N Fibonacci numbers.

This is done by pushing the integers 2, 3, … into a FibonacciProcessor that computes the 2nd, 3rd, … Fibonacci number.

On a machine with multiple cores, this program should run slower than FibonacciParallel for the same value of UPPER_BOUND. Note that the performance gain may become apparent only for a large value of this parameter. If only a small number of events are pushed the overhead of using multiple threads offsets the potential performance gains offered by concurrency.
